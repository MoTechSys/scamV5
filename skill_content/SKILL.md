# مهارة مشروع S-ACM

هذه المهارة توفر دليلاً شاملاً لتطوير وصيانة مشروع S-ACM. تحتوي على جميع المعلومات اللازمة لفهم بنية المشروع، القرارات التقنية، والمتطلبات الوظيفية.

## 1. نظرة عامة على المشروع

S-ACM هو نظام إدارة محتوى أكاديمي ذكي مبني باستخدام Django. يهدف إلى توفير منصة مركزية للطلاب والمدرسين والمسؤولين لإدارة المقررات الدراسية، المحتوى التعليمي، والتواصل. يتميز النظام بتكامل مع Google Gemini AI لتقديم ميزات ذكية مثل تلخيص المحتوى وتوليد الأسئلة.

## 2. الهيكل العام للمشروع

```
s-acm-project/
├── SKILL.md                # هذا الملف
├── references/             # وثائق تفصيلية
│   ├── project_vision.md   # الرؤية الكاملة والمتطلبات
│   ├── architecture.md     # القرارات المعمارية والتقنية
│   └── ui_design.md        # تحليل رسومات الواجهات
├── templates/              # قوالب الواجهة الأمامية
│   ├── layouts/
│   ├── components/
│   └── ...
└── scripts/                # سكريبتات مساعدة
    └── setup_dev.sh        # إعداد بيئة التطوير
```

## 3. خطوات العمل الرئيسية

### أ. فهم المشروع (للمطورين الجدد)

1.  **اقرأ وثيقة الرؤية**: ابدأ بقراءة `/home/ubuntu/skills/s-acm-project/references/project_vision.md` لفهم الأهداف والمتطلبات.
2.  **اقرأ وثيقة المعمارية**: انتقل إلى `/home/ubuntu/skills/s-acm-project/references/architecture.md` لفهم البنية التقنية.
3.  **اقرأ وثيقة التصميم**: اطلع على `/home/ubuntu/skills/s-acm-project/references/ui_design.md` لفهم تصميم الواجهات.

### ب. إعداد بيئة التطوير

- قم بتشغيل السكريبت التالي لإعداد بيئة التطوير وتثبيت المتطلبات:
  ```bash
  bash /home/ubuntu/skills/s-acm-project/scripts/setup_dev.sh
  ```

### ج. تنفيذ التعديلات (إعادة هيكلة الواجهة الأمامية)

- اتبع الخطة التفصيلية الموجودة في `references/architecture.md` لإعادة هيكلة الواجهة الأمامية.
- استخدم القوالب الموجودة في `templates/` كنقطة بداية.

### د. إضافة ميزات جديدة

1.  **تحديث الصلاحيات**: تأكد من إضافة أي صلاحيات جديدة مطلوبة في `apps/accounts/models.py`.
2.  **إنشاء الـ Views والـ URLs**: اتبع نمط التصميم الحالي.
3.  **إنشاء القوالب**: استخدم المكونات الموجودة في `templates/components/` قدر الإمكان.
4.  **تحديث القائمة الجانبية**: قم بتحديث `apps/core/services.py` لإضافة الرابط الجديد إلى القائمة الجانبية.

## 4. الموارد المرفقة

- **`references/`**: تحتوي على جميع الوثائق التفصيلية للمشروع.
- **`templates/`**: تحتوي على هيكل الواجهة الأمامية المقترح والمكونات القابلة لإعادة الاستخدام.
- **`scripts/`**: تحتوي على سكريبتات مساعدة لأتمتة المهام المتكررة.
